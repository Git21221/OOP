What is Class?
Class is a collection of objects and it doesn't take any memory. Class is also called as blueprint / logical entity.
What is Object?
Object is an instance of class that executes the class. Once the object is created, it takes up space like other variable in memory.
What is Constructor? full explanation.
Constructor is a speacial type of method whose name is same as class name. N.B:
  1. The main purpose of constructor is initilize the object.
  2. Every java class has a constructor.
  3. A constructor is autometically called at the time of object creation.
  4. A constructor doesn't have any return type including void.
Types of Constructor:
  1. private
  2. default
  3. Parameterized
  4. Copy

Inheritance

What is inheritance? Full explanation.
When we construct a new class from existing class in such a way that the new class access all the features & properties of existing class is called inheritance.
N.B:
  1. In, java extends keyword is used to perform inheritance.
  2. It provides code reusability.
  3. We can't access private members of class through inheritance.
  4. A subclass contains all the features of super class so, we should create the object of subclass.
  5. Method overriding only possible through inheritance.
  class A{
    //super class
  }
  class B extends A{
    //subclass
  }
Types of inheritance:
  1. Single Inheritance
  2. Multilevel inheritance
  3. Multiple inheritance
  4. Hierarchial inheritance
What is super keyword?
Super keyword refers to the objects of super class, it is used when we want to call the super class variable, method & constructor thorugh subclass.
N.B:
  1. Whenever the super class & subclass variable and method name both are same then it can be used only.
  2. To avoid the confusion between super and subclass variable and method. 
    class A{
      int a = 10;
    }
    class B extends A{
      super();
      int a = 20;
      void show(){
        System.out.print(a);
        System.out.print(super.a);
      }
    }
    class Test{
      public static void main(String[] args){
        B r = new B();
        r.show();
      }
    }
What is "this" keyword?
  1. "this" keyword refers to the current object inside a method or constructor.
  2. Whenever the name of instance and local variable are same the our runtime environment JVM gets consfused.
  3. It is also used When we want to call the default constructor of it's own class.
  4. It also call parameterized constructor of its own class
    class A{
      int a; //instance
      A(int a){
        this.a = a;

Polymorphism


Polymorphism is a geek word that means "same object having different behaviour".
Types of Polymorphism:
i) Compiletime Polymorphism (Early binding)
ii) Runtime Polymorphism (Late binding)
Compile Time Polymorphism
A Polymorphism which exists at the time of compilation is called Compile Time Polymorphism or early binding or static binding.
Example: Method overloading.
Whenever a class contain more than one method with same name and different types of parameter called method overloading.
Runtime Polymorphism
A Polymorphism which exists at the time of execution of program is called runtime Polymorphism.
Example: Method overriding
Whenever we write method in super and subclass in such a way that name and parameter must be same then its a method overriding.

N.B:

Abstraction
Abstraction is a process of hiding the implementation from the user, only the highlighted set of services provided to the user. Advantage:
  1. Security
  2. Enhancement
Two type of implementation of abstraction
  1. Abstract class
  2. Interface
What is abstract class?
A class which contains the abstract keyword in its declaration is called abstarct class.
N.B:
  1. We can't create object for abstract class.
  2. It can have abstract and non-abstract methods.
  3. To use an abstract class you have to inherit it from another subclass.
  4. If a class contains partial implementation then we should declare a class as abstarct.
What is abstract method?
A method which contains abstract modifier at the time of declaration is called abstract method.
N.B:
  1. It can only be used in abstract class.
  2. It doesn't contain any body "{}" and always ends with ";".
  3. Abstract method must be overriden in subclasses otherwise it will be also become a abstarct class.
  4. Whenever the action is common but implementations are different then we should use abstract method.
What is Interface?
Interface is just like a class, which contains only abstract method. To achieve interface java provides a keyword called 'implements'.
N.B:
  1. interface methods are default public & asbtract.
  2. Interface variable are by default public + static + final
  3. interface method must be overriden inside the implementing class.
  4. Interface nothing but deals between client & developer
Multiple inheritance using interface?
we can achieve multiple inheritace through interfaces because interface contains only abstract method, which implementation is provided by the subclasses.

Thread scheduler is the part of JVM which executes multiple threads on a single processor randomly.
What is thread life cycle?
As we know a thread is well known for independent execution. During the life cycle a thread can move from different state. i) New state (Born) ii) Runnable state (Ready) iii) Running state (Execution) iv) Waiting state (Blocked) v) Dead state (Exit)
